# Health Status Message Contract

## Overview
The `/health_status` topic provides real-time system health metrics for Physical AI hardware components.

## Message Definition
```
# HealthStatus.msg
string node_name
float64 timestamp
float64 cpu_percent
float64 memory_percent
float64 disk_percent
float64 temperature
string status
builtin_interfaces/Time header
```

## Fields

| Field | Type | Description | Validation |
|-------|------|-------------|------------|
| node_name | string | Name of the monitoring node | Required, non-empty |
| timestamp | float64 | Unix timestamp of measurement | Current time (±5s) |
| cpu_percent | float64 | CPU usage percentage | 0.0 ≤ value ≤ 100.0 |
| memory_percent | float64 | Memory usage percentage | 0.0 ≤ value ≤ 100.0 |
| disk_percent | float64 | Disk usage percentage | 0.0 ≤ value ≤ 100.0 |
| temperature | float64 | Temperature in Celsius | ≥ -273.15 (or -1 if unavailable) |
| status | string | Overall health status | "OK", "WARNING", or "CRITICAL" |
| header | builtin_interfaces/Time | Standard ROS2 header | Auto-generated by ROS2 |

## Example Message
```
node_name: "jetson_nano_01"
timestamp: 1702285200.123
cpu_percent: 45.2
memory_percent: 62.8
disk_percent: 23.1
temperature: 42.5
status: "OK"
header:
  stamp:
    sec: 1702285200
    nanosec: 123000000
  frame_id: ""
```

## Publishing Frequency
- Default: Once per second (1 Hz)
- Configurable via `publish_frequency` parameter
- Minimum: Every 0.1 seconds (10 Hz)
- Maximum: Every 60 seconds (1/60 Hz)

## Expected Subscribers
- Dashboard services
- Alert processing nodes
- Data logging services
- Operator interfaces

## Error Handling
- If a metric cannot be obtained, it should be set to -1.0
- The node should continue operating even if some metrics are unavailable
- Errors should be logged but not published as part of the health status
